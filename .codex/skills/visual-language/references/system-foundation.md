# System Foundation

A good deck visual language starts at the system level, not the slide level.

## What Must Be Stable Across The Deck

Lock these before styling individual slides:
- palette roles
- typography scale
- spacing rhythm
- section structure
- recurring layout families
- default treatment for evidence, warnings, comparisons, and transitions

## Palette Roles

Use color semantically, not decoratively.

Recommended roles:
- primary actor: the main system, method, or thesis thread
- secondary actor: baseline, alternative, or supporting process
- neutral structure: containers, scaffolding, separators, inactive states
- evidence: quantitative proof, numeric outcomes, experimental support
- warning: limitations, bottlenecks, failure cases
- success: strong positive outcome or validated claim
- low-emphasis support: context that should remain visible but quiet

Rules:
- keep the number of high-saturation colors small
- do not reassign the same role to different meanings on different slides
- reserve the strongest contrast for the current message, not for decorative accents

## Type Scale

Each deck should use a small, predictable set of text roles:
- title
- section marker
- main claim
- body/supporting text
- caption/source
- numeric highlight

Rules:
- the same text role should look the same everywhere
- claim text should be visually distinct from body text
- avoid introducing a new type treatment for a one-off special slide

## Spacing Rhythm

Define a reusable spacing system, such as small / medium / large.

Rules:
- use spacing to express grouping and separation before using lines or boxes
- repeated slide families should reuse the same internal gaps
- if a slide only works with many one-off offsets, the system is probably underspecified

## Layout Families

Choose 2-4 recurring slide skeletons and reuse them.

Examples:
- title + evidence
- side-by-side comparison
- process diagram + interpretation
- big takeaway + supporting proof

Why:
- repeated structures reduce cognitive load
- consistent skeletons make deviations meaningful instead of accidental
