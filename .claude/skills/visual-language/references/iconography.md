# Iconography And Symbol Language

Use a small, stable symbol vocabulary so the audience learns the visual language quickly.

## Core Principle

The same concept should keep the same visual metaphor throughout the deck.

Do not represent a dataset as a card on one slide, a cylinder on the next, and an icon on the third unless the meaning truly changes.

## Suggested Symbol Roles

| Meaning | Preferred family |
|---------|------------------|
| Actor / system | rounded container or labeled icon |
| Data / artifact | neutral card or document-like block |
| Process / transform | directional node with clear in/out flow |
| Evidence / metric | numeric block, chart callout, or highlighted result card |
| Warning / limitation | outlined callout or warm accent badge |
| Transition / bridge | thin section card, divider, or restrained connector |

## Icon Rules

- reduce each icon to its core meaning; remove decorative detail
- keep stroke weight, fill style, corner logic, and complexity level consistent
- if icons are outline-based, do not mix them casually with heavy filled icons
- if a concept appears on multiple slides, reuse the same icon or same shape family
- modify state with fill, border, or label before inventing a new icon metaphor

## Connector Rules

- solid arrows for causal or process flow
- thin lines for association only
- reserve accent color for the connector that carries the main story path
- do not mix multiple arrow grammars on the same slide unless they encode different meanings

## Complexity Control

- use as few symbol families as the talk can support
- if a viewer must learn a legend just to parse one slide, simplify the symbol system
- a new symbol family should only be introduced when a new semantic role appears
