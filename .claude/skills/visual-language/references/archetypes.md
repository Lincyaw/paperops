# Visual Archetypes

Use these archetypes as starting points, not as rigid themes. Pick one when the user wants a coherent deck style quickly, then adapt it into a full style brief.

## Archetype 1: Conference Minimal

Best for:
- 8-20 minute conference talks
- paper presentations where the audience wants results quickly
- technical rooms that reward fast parsing over atmosphere

Keywords:
- compressed
- precise
- high-signal
- restrained
- evidence-first

System defaults:
- palette roles: one cool primary, one muted support color, one warm accent reserved for key result or warning
- type scale: compact titles, strong claim line, minimal body text
- spacing rhythm: tight but regular; use whitespace surgically to isolate the main claim
- layout families: title + evidence, side-by-side comparison, result highlight, backup detail

Symbol language:
- use simple rounded containers and minimal line icons
- reserve badges for result state or benchmark labels only
- keep connectors thin and direct; prefer straight paths over decorative curves

Image policy:
- prefer plots, tables, and compact native diagrams
- screenshots only when the interface itself is the evidence
- avoid photos unless they are central to understanding the setup

Emphasis rules:
- one dominant proof point per slide
- let contrast and placement do most of the work
- use accent color sparingly so the strongest result feels earned

Failure mode:
- can become cold or generic if every slide uses the same skeletal structure without one or two memorable anchors

## Archetype 2: Research Seminar

Best for:
- internal seminars
- invited research talks with more context and explanation
- adjacent-field audiences who need stronger onboarding

Keywords:
- explanatory
- structured
- calm
- scholarly
- cumulative

System defaults:
- palette roles: slightly warmer neutral structure, one stable primary actor color, one softer evidence color, restrained warning color
- type scale: generous headings, readable body, clear section markers
- spacing rhythm: medium to loose; groups should breathe
- layout families: concept build-up, process + interpretation, case-study panel, synthesis slide

Symbol language:
- use clear card structures and repeated section dividers
- process symbols can be a bit more explicit than in conference mode
- use callouts for intuition and interpretation, not only for final results

Image policy:
- native diagrams and annotated screenshots are common
- plots still matter, but should sit inside stronger framing and interpretation
- avoid decorative stock imagery

Emphasis rules:
- hierarchy should guide the audience from framing -> mechanism -> evidence
- repetition is valuable; recurring slide skeletons help the room stay oriented
- transitions should feel visible but not theatrical

Failure mode:
- can drift into lecture-mode density if too many support elements are given equal weight

## Archetype 3: Bold Systems

Best for:
- systems, product, or lab-vision talks that need a distinctive identity
- job talks or flagship talks where memorability matters
- decks that must present a research agenda, not just a single result

Keywords:
- confident
- modular
- architectural
- memorable
- directional

System defaults:
- palette roles: one dominant primary, one dark structural neutral, one vivid accent for forward motion, one controlled warning color
- type scale: strong section cards, prominent numeric highlights, compact supporting text
- spacing rhythm: medium to large; strong blocks and deliberate negative space
- layout families: agenda card, architecture spread, milestone comparison, thesis + evidence slide

Symbol language:
- use a small number of strong shape families for systems, data, and outcomes
- connectors should visibly encode flow and dependency
- repeated motif can be slightly bolder, but must remain disciplined across slides

Image policy:
- native architecture diagrams and annotated system visuals are preferred
- photos are rare; if used, they should reinforce the lab or real-world stakes
- charts should be visually integrated into the same system rather than dropped in as unrelated artifacts

Emphasis rules:
- focal points can be stronger than in the other archetypes, but only one per reveal
- repetition + intentional breaks should create memorability
- accent color and large blocks should mark thesis-level moments, not ordinary labels

Failure mode:
- can become visually loud if accent color, thick borders, and oversized callouts appear everywhere

## Selection Rule

Choose the archetype that best matches audience distance, venue tempo, and how memorable versus restrained the deck should feel. Then convert it into a concrete brief with `style-brief-template.md` rather than copying it literally.
